# AD9910 驱动模块移植说明文档

本模块适用于基于 STM32 HAL 库的 AD9910 高速 DDS 芯片驱动。

## 1. 硬件连接与引脚定义

在移植前，请务必根据目标板卡的实际电路，在 `AD9910.h` 中修改以下宏定义。

## 关键控制组 (GPIO 映射)

| **功能分类** | **驱动宏定义**    | **推荐端口**  | **说明**                 |
| ------------ | ----------------- | ------------- | ------------------------ |
| **SPI 通信** | `AD9910_CS_Pin`   | GPIOE/Pin 3   | 片选信号 (低电平有效)    |
|              | `AD9910_SCLK_Pin` | GPIOE/Pin 2   | 串行时钟                 |
|              | `AD9910_SDIO_Pin` | GPIOE/Pin 5   | 串行数据输入             |
| **基础控制** | `AD9910_RST_Pin`  | GPIOE/Pin 1   | 全局复位 (Master Reset)  |
|              | `AD9910_IOUP_Pin` | GPIOD/Pin 5   | 寄存器更新触发信号       |
| **功能控制** | `AD9910_DRC_Pin`  | GPIOD/Pin 7   | 数字扫描方向控制 (DRCTL) |
|              | `AD9910_PF0~2`    | GPIOB/Pin 3~5 | Profile 选择引脚         |

## 2. 软件移植步骤

1. **添加文件**：将 `AD9910.h` 和 `AD9910.c` 添加到工程目录中。
2. **包含头文件**：在 `main.c` 或需要调用的地方添加 `#include "AD9910.h"`。
3. **GPIO 初始化**：确保在调用驱动前，已通过 CubeMX 生成或手动调用 `MX_GPIO_Init()`，且所有引脚配置为 **推挽输出 (Output Push-Pull)**。
4. **系统时钟校准**：
   - AD9910 的频率计算依赖于系统主频（SYSCLK）。
   - 驱动中默认系数 $4.294967296$ 是基于模块**1GHz** 主频计算的 ($2^{32} / 10^9$)。

## 3. 核心 API 使用指南

## 3.1 基础频率输出

C

```
Init_AD9910();                 // 1. 初始化寄存器配置
AD9910_FreWrite(10000000);     // 2. 输出 10MHz 正弦波
AD9910_AmpWrite(16383);        // 3. 设置为最大幅度 (14-bit)
```

## 3.2 频率扫描 (DRG 模式)

使用 `AD9910_DRG_FrePara_Set` 配置扫描参数：

- **lowFre / upFre**: 扫描的上下限频率。
- **posStep / negStep**: 步进的频率分辨率。
- **posRate / negRate**: 步进的时间间隔。

## 3.3 RAM 波形模式

调用 `AD9910_RAM_WAVE_Set(WAVE_TYPE)` 可以切换预设的波形（如三角波、方波、Sinc波）。

## 4. 注意事项（必读）

- **IO_UPDATE 脉冲**：所有的寄存器写入操作后，必须通过 `AD9910_UP_DAT_1` 产生一个上升沿，配置才会真正生效。
- **SPI 时序**：本驱动使用软件模拟 SPI。如果你的 MCU 主频极高（如 H7 系列），可能需要在 `txd_8bit` 函数中添加微小的延时以满足 AD9910 的建立时间要求。
- **寄存器冲突**：启用 DRG 扫描时，请确保 CFR2 寄存器中的扫描使能位已正确配置，且引脚 `DRHOLD` 保持低电平。
