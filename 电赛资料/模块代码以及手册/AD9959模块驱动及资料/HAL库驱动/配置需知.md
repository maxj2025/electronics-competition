## AD9959 驱动移植说明文档

## 1. 硬件连接与配置要求

AD9959 是一款 4 通道、500 MSPS 的直接数字频率合成器（DDS）。该代码采用**模拟 SPI（软件 IO 翻转）**方式进行通信。

#### **GPIO 需求清单**

你需要使用 CubeMX 生成的代码中定义以下引脚（与 `AD9959.h` 中的宏定义对应）：

- **控制引脚**：
  - `AD9959_CS`: 片选信号（低电平有效）。
  - `AD9959_SCLK`: 串行时钟。
  - `AD9959_UPDATE`: I/O 更新信号（上升沿将数据从缓冲寄存器传送到活动寄存器）。
  - `AD9959_RESET`: 硬件复位引脚。
  - `AD9959_PDC`: 电源向下控制（通常接低电平以正常工作）。
- **数据引脚**：
  - `AD9959_SDIO0`: 数据线 0（主要写入引脚）。
  - `AD9959_SDIO1/2/3`: 在 2/4 线模式或调制模式下使用（P0-P3 功能）。
- **功能引脚 (Profile/Modulation)**：
  - `AD9959_SP0` ~ `AD9959_SP3`: 用于频率/相位/幅度调制的切换引脚。
  - 简而言之就是开启13个GPIO，在CubeMx里面自定义引脚名称与上述名称一致。

## 2. 软件移植步骤

#### **Step 1: 适配底层 HAL 库**

代码目前是基于 **STM32H7** 系列编写的。如果你使用 F1、F4 或其他系列：

1. 修改 `AD9959.h` 中的 `#include "stm32h7xx.h"` 为你对应的系列头文件。
2. 确保所有的 `HAL_GPIO_WritePin` 调用与你的库兼容。

#### **Step 2: 完善 `IO_Update` 函数**

AD9959 的所有寄存器写入后，必须通过 `UPDATE` 引脚的一个高脉冲才会生效。

- 如果你的 CPU 频率极高（如 H7, 480MHz+），当前的 `delay1(2)` 已经足够。
- 如果发现设置不生效，请检查 `IO_Update` 函数产生的脉冲宽度是否满足手册要求的最小 $1$ 个系统时钟周期。

#### **Step 3: 频率计算因子核对**

代码中定义了：

```
double ACC_FRE_FACTOR = 8.59;
```

这是基于 $2^{32} / 500,000,000$ 计算得来的。

> **公式**：
>
> $$FTW = \frac{Desired Frequency \times 2^{32}}{SYSCLK}$$
>
> 如果你更改了 AD9959 的系统时钟，请务必重新计算此参数。

## 3. 注意事项 (Critical)

1. **单通道与多通道选择**：

   AD9959 在操作特定通道前，必须先写 **CSR (Channel Select Register)**。代码中的 `AD9959_Set_Fre` 等函数已经包含了这一步。例如，如果你只想操作通道 0，传入的 `Channel` 应该是 `CH0` (0x10)。

2. **调制模式的引脚物理限制**：

   - **2电平调制**：P0-P3 引脚分别对应控制 CH0-CH3。
   - **4/8/16电平调制**：由于引脚有限，高阶调制会占用多个引脚，导致无法同时独立控制 4 个通道。例如，4电平调制时只能控制 2 个通道。

3. **IO 模式同步**：

   AD9959 默认是 1 线串行模式（SDIO0 写入）。本代码采用的就是此模式。如果你的硬件电路设计为 2 线或 4 线并写，需修改 `FR1` 寄存器的配置。

4. **扫频/扫幅时间计算**：

   `rsrr`（上升斜率）和 `fsrr`（下降斜率）的单位是系统时钟的 4 倍。

   $dT = \text{控制字} \times (4 / \text{SYSCLK})$。

   在 500 MHz 时钟下，控制字 1 约等于 $8\text{ns}$。

------

## 4. 推荐初始化流程

在你的 `main.c` 中，建议按以下顺序调用：

C

```
// 1. 系统外设初始化 (GPIO等)
// 2. AD9959 初始化
AD9959_Init();

// 3. 设置一个固定频率测试 (通道0, 10MHz, 最大幅度, 0相位)
AD9959_setsine(CH0, 10000000, 1023, 0);

// 4. 必须手动调用一次更新脉冲（如果函数内部没调的话）
IO_Update();
```
