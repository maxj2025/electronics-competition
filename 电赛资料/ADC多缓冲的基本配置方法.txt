DMA多缓冲区的配置（ADC）

调用 HAL_DMAEx_MultiBufferStart_IT开启多缓冲模式，此为DMA单独搬运数据ADC_DR寄存器，未与ADC绑定

ADC并未在初始配置时将ADC_CR2寄存器的DMA位置1（ADC），所以运行时DR会溢出，于是把ADC_SR寄存器OVR位置1，停用一切DMA

因此在ADC配置时需要手动开启CR2寄存器的DMA位，
LL库开启DMA如下：LL_ADC_REG_SetDMATransfer(ADC1, LL_ADC_REG_DMA_TRANSFER_UNLIMITED);

或	

hadc1.Instance->CR2 |=ADC_CR2_DMA;

注意：调用 HAL_DMAEx_MultiBufferStart_IT开启多缓冲模式时，函数内写明需要注册回调函数，否则不正常配置，至于为什么HAL库不把它弄得像其它回调写成weak函数而是要传函数指针，因为DMA与多个外设都能配合，不想让所有的DMA中断都进入同一个回调显得代码及其臃肿，所以需要手动注册

示例：先在PV声明
/* USER CODE BEGIN PV */

void DMA_XferCpltCallback(DMA_HandleTypeDef *hdma);
void DMA_XferM1CpltCallback(DMA_HandleTypeDef *hdma);
void DMA_XferErrorCallback(DMA_HandleTypeDef *hdma);

/* USER CODE END PV */

主函数里传参（传的是函数指针）
/* USER CODE BEGIN WHILE */
hdma_adc1.XferCpltCallback  =  DMA_XferCpltCallback;    // 缓冲区0完成回调
hdma_adc1.XferM1CpltCallback = DMA_XferM1CpltCallback;   // 缓冲区1完成回调
hdma_adc1.XferErrorCallback = DMA_XferErrorCallback;     // 错误回调

回调函数的书写

void DMA_XferCpltCallback(DMA_HandleTypeDef *hdma)
{
  if (hdma == &hdma_adc1)  // 确认是ADC的DMA中断
 	{
  		test_num1++;
 	}
}

// 2. 缓冲区1（adc_buffer[1]）传输完成回调
void DMA_XferM1CpltCallback(DMA_HandleTypeDef *hdma)
{
  if (hdma == &hdma_adc1)
 	 {
		test_num2++;
 	 }
}

// 3. DMA传输错误回调
void DMA_XferErrorCallback(DMA_HandleTypeDef *hdma)
{
  if (hdma == &hdma_adc1)
 	 {

  	}
}
